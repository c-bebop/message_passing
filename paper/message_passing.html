Message Oriented Communication

There are several mechanisms of communication in distributed systems, such as remote procedure calls and remote object invocations. Both mechanisms rely on direct processing after the request. This leads to problems if the component A relies on component B to process the request, but component B is not able to process in time Tanenbaum and Steen (2007).

One of the most common abstract architecture is the Message-Oriented Transient Communication . TCP for example does provide all the neeeded socket primitives to build a sytem for developers, needing message based communication Tanenbaum and Steen (2007).

Message-Passing Interface (MPI)

The Massage-Passing Interface (MPI) is another Message Passing standard, made for high performance and scale able distributed systems where high-efficiency is needed. The MPI standard was designed by the MPI Forum which is an open group with representatives from many organizations Message Passing Interface Forum (2012).

One goal of the MPI is to simplify the communication primitives and bring them to an abstraction level to perfectly fit the programmers needs of writing efficient and clean code for such distributed systems Tanenbaum and Steen (2007).

TCP does not fit those requirements. While the socket primitives read and write are sufficient for several general-purpose protocols, (managing the communication across networks) they are insufficient for high-speed interconnection networks such as super computers or server clusters . With the possibility of implementing proprietary protocols, additional primitives can simplify and speed up the communication via messages Tanenbaum and Steen (2007).

Message Passing in Erlang

Erlang is a functional programming language that was written for the need of real-time, non-stop, concurrent, very large and distributed system applications Armstrong (1996). While the language was originally designed by Joe Armstrong, Robert Virding and Mike Williams for Ericsson (a Swedish telecommunications provider) in 1986, it finally became open source in 1998, thanks to the open source initiatives lead by Linux Däcker (2000).

Since there is a lot confusion with the naming of Erlang: Joe Armstrong, who started to design Erlang by adding functionality to Prolog, named it after the Danish mathematician Agner Krarup Erlang (creator of the Erlang loss formula) following the tradition of naming programming languages after dead mathematicians Däcker (2000).

Erlang uses native Message Passing to communicate between light weight processes in distributed systems Armstrong (1996).
